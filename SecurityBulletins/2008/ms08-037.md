# Microsoft Security Bulletin MS08-037 - Important

## Sikkerhetsproblemer i DNS kan tillate etterligningsangrep (953230)

Publisert: 8. juli 2008 | Oppdatert: 13. januar 2009

**Versjon:** 2.3

### Generell informasjon

#### Kortfattet sammendrag

Denne sikkerhetsoppdateringen løser to personlig rapporterte sikkerhetsproblemer i DNS (Domain Name System) i Windows som kan tillate etterligningsangrep. Disse sikkerhetsproblemene finnes i både DNS-klienten og DNS-serveren og kan tillate at en ekstern angriper omadresserer nettverkstrafikk som er ment for systemer på Internett, til angriperens egne systemer.

Denne sikkerhetsoppdateringen har alvorlighetsgraden Viktig for alle støttede versjoner av Microsoft Windows 2000, Windows XP, Windows Server 2003 og Windows Server 2008. Hvis du vil ha mer informasjon, kan du se avsnittet **Berørt og uberørt programvare** i denne delen.

Sikkerhetsoppdateringen retter opp sikkerhetsproblemene ved hjelp av svært tilfeldige DNS-transaksjons-IDer, ved å bruke tilfeldige socketer for UDP-spørringer og ved å oppdatere logikken som brukes til å administrere DNS-hurtigbufferen. Hvis du vil ha mer informasjon om sikkerhetsproblemet, kan du se avsnittet Vanlige spørsmål for den bestemte sikkerhetsproblemoppføringen under neste del, **Informasjon om sikkerhetsproblemer**.

**Anbefaling.** Microsoft anbefaler at kundene installerer oppdateringen så snart det er mulig.

**Kjente problemer.** [Microsoft Knowledge Base-artikkel 953230](http://support.microsoft.com/kb/953230) dokumenterer kjente problemer som kundene kan oppleve når de installerer denne sikkerhetsoppdateringen.

## Vanlige spørsmål som er relatert til denne sikkerhetsoppdateringen

**Hvorfor ble denne sikkerhetsbulletinen revidert 13. januar 2009?**  
Microsoft reviderte denne bulletinen for å informere om at oppdateringen som tilbys av denne bulletinen, kanskje ikke har blitt tilbudt på riktig måte til alle systemer som kjører Windows XP Service Pack 3. Problemet med gjenkjenning og distribusjon har blitt løst, og kunder med Windows XP Service Pack 3-systemer som ikke allerede har tatt i bruk oppdateringene i denne bulletinen, vil nå bli tilbudt oppdateringen på riktig måte. Det var ingen endringer i binære filer for denne oppdateringen. Kunder som har oppdatert systemene sine, trenger ikke å installere denne oppdateringen på nytt.

**Hvorfor ble denne sikkerhetsbulletinen revidert 25. juli 2008?**  
Microsoft reviderte denne sikkerhetsbulletinen for å legge til tre nye kjente problemer under **Vanlige spørsmål som er relatert til denne sikkerhetsoppdateringen**. Disse vanlige spørsmålene er som følger:

  - **Hva består portkonfliktproblemene av, som kunder kan oppleve etter at denne oppdateringen er installert?**  
    DNS-servere som utfører ytterligere tjenester som krever statiske UDP-porter i portområdet som DNS-serveren har tildelt socketutvalget, kan oppleve portkonflikter som fører til tjenestefeil. Hvis du vil ha mer informasjon, kan du se [Microsoft Knowledge Base-artikkel 953230](http://support.microsoft.com/kb/953230) som dokumenterer kjente problemer som kundene kan oppleve når de installerer denne sikkerhetsoppdateringen.
  - **Hva består problemene av, som SBS-kunder kan oppleve etter at denne oppdateringen er installert?**  
    Støttede versjoner av Windows Small Business Server (SBS) 2003 inneholder samme berørte kode som Windows Server 2003 Service Pack 1 og Windows Server 2003 Service Pack 2. SBS-systemer kan imidlertid støte oppleve tjenestefeil eller at de ikke fungerer skikkelig etter at sikkerhetsoppdateringen for DNS-serveren som tilbys i denne bulletinen, er installert. Hvis du vil ha mer informasjon, kan du se [Microsoft Knowledge Base-artikkel 953230](http://support.microsoft.com/kb/953230) som dokumenterer kjente problemer som kundene kan oppleve når de installerer denne sikkerhetsoppdateringen.
  - **Hva består problemene av, som perimeterbrannmurer og perimeter-NAT-enheter kan oppleve etter at denne oppdateringen er installert?**  
    Når sikkerhetsoppdateringen som tilbys av denne bulletinen, er installert på en Windows-basert datamaskin, bruker DNS-spørringer fra den verten på tvers av en brannmur ikke lenger vilkårlige kildeporter. Hvis du vil ha mer informasjon, kan du se [Microsoft Knowledge Base-artikkel 953230](http://support.microsoft.com/kb/953230) som dokumenterer kjente problemer som kundene kan oppleve når de installerer denne sikkerhetsoppdateringen.

**Hvorfor ble denne sikkerhetsbulletinen revidert 23. juli 2008?**  
Microsoft reviderte denne sikkerhetsbulletinen for å føre opp MS06-064, MS07-062 og MS08-001 som bulletiner som erstattes av denne oppdateringen.

**Hvorfor ble denne sikkerhetsbulletinen revidert 10. juli 2008?**  
Microsoft reviderte denne sikkerhetsbulletinen for å informere brukere av ZoneAlarm og Check Point Endpoint Security (tidligere kjent som Check Point Integrity), fra [Check Point Software Technologies Ltd.](http://www.checkpoint.com/), om et problem med Internett-tilkoblingen beskrevet i følgende vanlige spørsmål. Revisjonen endret ikke filene i sikkerhetsoppdateringen i denne bulletinen. Kunder som allerede har installert denne oppdateringen, trenger ikke å gjøre det på nytt. Brukere av programvaren fra Check Point nevnt ovenfor bør imidlertid lese følgende vanlige spørsmål for videre veiledning.

**Hva er problemet som brukere av ZoneAlarm og Check Point Endpoint Security kan oppleve etter at de har installert denne oppdateringen?**  
Microsoft er oppmerksom på rapporter om at brukere av ZoneAlarm og Check Point Endpoint Security (tidligere kjent som Check Point Integrity), fra Check Point Software Technologies Ltd., opplever et problem med Internett-tilkoblingen etter å ha installert sikkerhetsoppdateringene som tilbys av denne sikkerhetsbulletinen, MS08-037. Microsoft fortsetter å undersøke dette problemet med Check Point. Microsoft oppfordrer brukere av [ZoneAlarm](http://download.zonealarm.com/bin/free/pressreleases/2008/lossofinternetaccessissue.html) og Check Point Endpoint Security til å se gjennom det aktuelle [webområdet for Check Point](https://supportcenter.checkpoint.com/supportcenter/index.jsp) og denne sikkerhetsbulletinen for den nyeste veiledningen eller programvareoppdateringer.

Microsoft gir deg kontaktinformasjon til tredjeparter slik at du kan få kundestøtte. Denne kontaktinformasjonen kan endres uten varsel. Microsoft garanterer ikke nøyaktigheten til kontaktinformasjonen til tredjepartene. Tredjepartsprodukter som beskrives i denne artikkelen, produseres av selskaper som er uavhengige av Microsoft. Microsoft gir ingen garantier, verken indirekte eller på annen måte, om ytelsen eller påliteligheten til disse produktene.

**Hvilke avinstalleringsproblemer kan kundene oppleve etter at de har installert disse sikkerhetsoppdateringene?**  
Støttede versjoner av Microsoft Windows 2000 og Windows 2003-systemer vil få tilbud om både DNS-serveroppdatering og DNS-klientoppdateringer. DNS-serveroppdateringen og DNS-klientoppdateringene deler binære filer og må avinstalleres i motsatt rekkefølge av den de ble installert i, for å unngå at de delte binære filene går tilbake til tidligere versjoner. [Microsoft Knowledge Base-artikkel 823836](http://support.microsoft.com/kb/823836) dokumenterer kjente problemer som kundene kan oppleve når de avinstallerer denne sikkerhetsoppdateringen.

**Hvilke problemer kan kundene oppleve i forbindelse med tilkobling med tilfeldige socketer når de installerer denne sikkerhetsoppdateringen?**  
Som standard benyttes et stort antall tilgjengelige socketer og entropier i DNS-oppdateringene som tilbys gjennom denne sikkerhetsbulletinen. Hvis brukeren imidlertid har definert portområder i innstillingen, tar oppdateringene hensyn til de brukerdefinerte innstillingene og tildeler bare de definerte socketene.

Socketområder kan defineres på følgende steder i registeret:  
  
**HKLM\\System\\CurrentControlSet\\Services\\DNS\\Parameters Reg key Name: SocketPoolSize**  
  
**Obs\!** DNS-tjenesten må startes på nytt for at disse endringene skal bli implementert.

**Hva skjer når jeg definerer et utvalgsområde for socket?**  
Det kan være nødvendig å definere socketområdet som DNS kan velge fra, for å unngå konflikt med andre programmer eller tjenester som har behov for samme socketutvalg i kommunikasjonen. Se [MaxUserPort](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/regentry/58791.mspx?mfr=true) og [Microsoft Knowledge Base-artikkel 812873](http://support.microsoft.com/kb/812873) hvis du vil ha mer informasjon om disse registernøkkelinnstillingene.

**Hvordan er operativsystemene forskjellige i forbindelse med definering av utvalgsområdene for socket?**  
MaxUserPort-registernøkkelen har en annen betydning i Windows Vista og Windows Server 2008 enn i Microsoft Windows Server 2000 og Windows Server 2003. [Microsoft Knowledge Base-artikkel 929851](http://support.microsoft.com/kb/929851) inneholder mer informasjon om endring i virkemåte for Windows Vista og Windows Server 2008.

I Microsoft Windows Server 2000 og Windows Server 2003 definerer MaxUserPort sluttpunktet i det dynamiske portområdet. Området starter med 1024 og fortsetter til den brukerdefinerte verdien i MaxUserPort-registernøkkelinnstillingen. Etter at du har installert oppdateringene som tilbys gjennom denne sikkerhetsbulletinen, vil standard virkemåte i Microsoft Windows Server 2000 og Windows Server 2003 være å tildele socketer tilfeldig fra portområdet 49152 til 65535. Hvis MaxUserPort-området er definert, tildeles porter tilfeldig fra 1024 til den definerte verdien i MaxUserPort-registernøkkelinnstillingen. Gå til [Microsoft Knowledge Base-artikkel 812873](http://support.microsoft.com/kb/812873) hvis du vil ha mer informasjon om hvordan du reserverer portområder i Microsoft Windows 2000 Server og Windows Server 2003.

I Windows Vista og Windows Server 2008 definerer MaxUserPort startpunktet til det dynamiske portområdet. Som standard er området i Windows Vista og Windows Server 2008 fra 49152 til 65535.

Filinformasjonsdetaljene finner du i [Microsoft Knowledge Base-artikkel 953230](http://support.microsoft.com/kb/953230).

#### Begrensende faktorer for DNS-sikkerhetsproblemet med utilstrekkelig socket-entropi i DNS – CVE-2008-1447

Begrensning viser til en innstilling, vanlig konfigurering eller generell god fremgangsmåte som finnes i standardtilstanden og kan redusere alvorlighetsgraden til utnyttelsen av et sikkerhetsproblem.

  - Kryptografiprotokoller som kjører over TCP- og IP-nivåene, som IPsec eller SSL/TLS, kan motvirke at en angriper får muligheten til å overvåke eller forstyrre omdirigert trafikk.

#### Midlertidige løsninger for DNS-sikkerhetsproblemet med utilstrekkelig socket-entropi i DNS – CVE-2008-1447

Midlertidige løsninger viser til en innstilling eller konfigurasjonsendring som ikke løser det underliggende sikkerhetsproblemet, men hjelper til med å blokkere kjente angrepsvektorer før du installerer oppdateringen.

#### Vanlige spørsmål om DNS-sikkerhetsproblemet med utilstrekkelig socket-entropi i DNS – CVE-2008-1447

**Hva er omfanget av sikkerhetsproblemet?**    
Det finnes et sikkerhetsproblem i Windows DNS-klienter og DNS-servere som kan tillate etterligningsangrep. En angriper som utnytter dette sikkerhetsproblemet kan sette inn tilfeldige adresser i DNS-hurtigbufferen.

**Hva forårsaker sikkerhetsproblemet?**    
Windows DNS-tjenesten i Windows DNS-klienten og DNS-serveren tilbyr ikke tilstrekkelig entropi under DNS-spørringer.

**Hva er DNS (Domain Name System)?**  
DNS (Domain Name System) er en av de bransjeledende pakkene av protokoller som omfatter TCP/IP. DNS implementeres ved å bruke to programvarekomponenter: DNS-serveren og DNS-klienten (eller løseren). Begge komponentene kjøres som bakgrunnstjenesteprogrammer. Nettverksressurser identifiseres av numeriske IP-adresser, men det er vanskelig for nettverksbrukere å huske disse IP-adressene. DNS-databasen inneholder oppføringer som tilordner brukervennlige alfanumeriske navn for nettverksressurser, som www.microsoft.com, til IP-adressene som brukes til kommunikasjon av disse ressursene. På denne måten fungerer DNS som en mnemoteknisk enhet og gjør det enklere å huske nettverksressurser for nettverksbrukere. Hvis du vil ha mer informasjon, og du vil vise logiske diagrammer som illustrerer hvordan DNS passer sammen med andre Windows-teknologier, kan du lese artikkelen [Hva er DNS](http://technet2.microsoft.com/windowsserver/en/library/ff937311-03ce-4d04-b72c-b39c4d51cb361033.mspx).

**Hva er DNS-hurtigbuffer?**  
DNS (Domain Name System) Caching Resolver Service er en tjeneste som lagrer svarene på DNS-spørringer, slik at DNS-serveren ikke mottar de samme spørringene gjentatte ganger. Hvis du vil ha mer informasjon, kan du lese Technet-artikkelen om [DNSCache](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/regentry/30643.mspx?mfr=true) eller [DNS Resolver Cache Service](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/cnet/cnbc_imp_qxht.mspx?mfr=true). Du kan lese Technet-artikkelen om [Attack detection](http://www.microsoft.com/technet/isa/2004/help/fw_alertattack.mspx?mfr=true) for mer informasjon om skader på DNS-hurtigbufferen.

**Hvordan kan en angriper utnytte dette sikkerhetsproblemet?**    
En angriper som har utnyttet dette sikkerhetsproblemet, kan sette inn tilfeldige adresser i DNS-hurtigbufferen, noe som også omtales som skade på DNS-hurtigbufferen.

**Hvordan kan en angriper utnytte sikkerhetsproblemet?**
En angriper kan sende bestemte spørringer til en sårbar DNS-server eller -klient og samtidig svare på en måte som gjør det mulig for angriperen å sette inn falske eller misvisende DNS-data. Angriperen kan deretter omdirigere Internett-trafikken fra legitime plasseringer til en adresse som angriperen har valgt.

**Hvilke systemer er hovedsakelig i faresonen på grunn av sikkerhetsproblemet?**    
Alle Windows-systemer som er koblet til et nettverk eller til Internett, er i faresonen. Windows Vista og Windows Server 2008 berøres ikke av dette sikkerhetsproblemet.

**Hva gjør oppdateringen?**
Oppdateringen fjerner dette sikkerhetsproblemet ved å bruke svært tilfeldige DNS-transaksjons-IDer og tilfeldige UDP-socketer for eksterne spørringer.

**Var dette sikkerhetsproblemet offentliggjort da denne sikkerhetsbulletinen ble publisert?**    
Nei. Microsoft mottok informasjon om dette sikkerhetsproblemet gjennom begrenset offentliggjøring.

**Hadde Microsoft mottatt rapporter om at dette sikkerhetsproblemet ble utnyttet, da denne sikkerhetsbulletinen ble publisert?**    
Nei. Microsoft hadde ikke mottatt informasjon som tydet på at dette sikkerhetsproblemet offentlig hadde blitt brukt til å angripe kunder, og hadde ikke sett noen eksempler på at konseptbeviskode var publisert da denne sikkerhetsbulletinen først ble publisert.

    Ettersom det finnes flere versjoner av Microsoft Windows, kan fremgangsmåten være annerledes for datamaskinen du bruker. I så fall ser du produktdokumentasjonen for å utføre trinnene.
