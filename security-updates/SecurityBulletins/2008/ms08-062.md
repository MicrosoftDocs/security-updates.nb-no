# Microsoft Security Bulletin MS08-062 - Important

## Sikkerhetsproblem i Internett-utskriftstjeneste i Windows kan tillate ekstern kjøring av kode (953155)

Publisert: 14. oktober 2008 | Oppdatert: 29. oktober 2008

**Versjon:** 2.2

### Generell informasjon

#### Kortfattet sammendrag

Denne oppdateringen løser et personlig rapportert sikkerhetsproblem i Internett-utskriftstjenesten i Windows som kan føre til ekstern kjøring av kode. En angriper som klarer å utnytte dette sikkerhetsproblemet, kan få full kontroll over et berørt system. En angriper kan deretter installere programmer, vise, endre eller slette data eller opprette nye kontoer.

Denne oppdateringen har alvorlighetsgraden Viktig for alle støttede versjoner av Microsoft Windows 2000, Windows XP, Windows Server 2003 og Windows Server 2008. Hvis du vil ha mer informasjon, kan du se avsnittet **Berørt og uberørt programvare** i denne delen.

Sikkerhetsoppdateringen løser sikkerhetsproblemet ved å endre måten som minne beregnes på i IPP-tjenesten (Internett-utskriftsprotokoll). Hvis du vil ha mer informasjon om sikkerhetsproblemet, kan du se avsnittet Vanlige spørsmål for den bestemte sikkerhetsproblemoppføringen under neste del, **Informasjon om sikkerhetsproblemer**.

**Anbefaling.** Microsoft anbefaler at kundene installerer oppdateringen så snart det er mulig.

**Kjente problemer.** Ingen

## Vanlige spørsmål som er relatert til denne sikkerhetsoppdateringen

**Hvorfor ble denne bulletinen revidert 29. oktober 2008?**    
Microsoft oppdaterte denne bulletinen for å opplyse kunder om at oppdateringspakkene for Windows Server 2008 for Itanium-based Systems og alle støttede versjoner av Windows Vista nå er tilgjengelige på Windows Update, Microsoft Update, Windows Software Update Services (WSUS), Systems Management Server og System Center Configuration Manager. Da denne sikkerhetsbulletinen først ble utgitt, var oppdateringene for disse plattformene bare tilgjengelig på Microsoft Download Center.

**Hvorfor ble denne bulletinen revidert 16. oktober 2008?**  
Microsoft reviderte bulletinen for å informere om at Internett-utskriftstjenesten for Windows kjører i konteksten til utskriftskøtjenesten, som kjører med systemrettigheter. En angriper som klarer å utnytte sikkerhetsproblemet som beskrives i denne sikkerhetsbulletinen, kan kjøre vilkårlig kode med systemrettigheter og få fullstendig kontroll over det berørte systemet.

**Hvorfor ble denne bulletinen revidert 15. oktober 2008?**  
Microsoft reviderte denne bulletinen for å fjerne alvorlighetsgraden for Windows Server 2008 for Itanium-based Systems. Microsoft oppdaterte også denne bulletinen for å klargjøre at oppdateringen for Windows Server 2008 for Itanium-based Systems er tilgjengelig via Microsoft Download Center og vil bli publisert via andre distribusjonskanaler så snart som mulig.

**Hvorfor er Windows Server 2008 for Itanium-based Systems og Windows** **Vista** **oppført i tabellen over berørt programvare hvis de ikke har noen alvorlighetsgrad?**  
Windows Server 2008 for Itanium-based Systems og alle støttede versjoner av Windows Vista inneholder den delte utskriftskøkomponenten som oppdateres av denne sikkerhetsoppdateringen. Fordi Internett-utskrift ikke støttes på Windows Server 2008 for Itanium-based Systems eller Windows Vista, berøres imidlertid ikke vektoren som kan utnyttes, som omtales i denne sikkerhetsbulletinen. Som en beskyttelse i dybden mot alle mulige nye vektorer som identifiseres i fremtiden, oppfordres kunder med Windows Server 2008 for Itanium-based Systems og Windows Vista til å ta i bruk sikkerhetsoppdateringen.

**Hvor finner jeg filinformasjonsdetaljene?**

## Sikkerhetsproblem med overflyt av heltall i IPP-tjenesten – CVE-2008-1446

Det finnes et sikkerhetsproblem som kan føre til ekstern kjøring av kode, på Windows-systemer som kjører Microsoft Internet Information Services (IIS) med Internett-utskriftstjenesten for Windows aktivert. Dette problemet kan tillate at en ekstern, godkjent angriper kan kjøre vilkårlig kode på et berørt system.

Hvis du vil se dette sikkerhetsproblemet som en standardoppføring i listen Common Vulnerabilities and Exposures, kan du se [CVE-2008-1446](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1446).

#### Begrensende faktorer for sikkerhetsproblemet med overflyt av heltall i IPP-tjenesten – CVE-2008-1446

Begrensning viser til en innstilling, vanlig konfigurering eller generell god fremgangsmåte som finnes i standardtilstanden og kan redusere alvorlighetsgraden til utnyttelsen av et sikkerhetsproblem. Følgende begrensende faktorer kan være nyttige i din situasjon:

  - Internett-utskriftsprotokollen (IPP) er aktivert som standard i IIS for Microsoft Windows 2000 og Windows XP. Standard filtillatelser på tvers av alle plattformer er imidlertid angitt slik at IPP bare er tilgjengelig for gruppen med godkjente brukere. På grunn av denne begrensningen vil ikke-godkjente brukere som standard ikke ha de nødvendige mappetillatelsene som er nødvendige for å utnytte dette sikkerhetsproblemet eksternt.
  - Installasjoner av IIS på Windows Server 2003 er ikke berørt av dette sikkerhetsproblemet når standardkonfigurasjonen brukes. Bare installasjoner av Windows Server 2003 som har aktivert ISAPI-filteret for IPP i IIS er berørt av sikkerhetsproblemet. Både komponentene for Internett-utskrift og Active Server Pages er nødvendige for å kunne aktivere webbasert utskriftsadministrasjon og utskrift over Internett. Hvis komponenten for Active Server Pages er satt til Tillatt og komponenten for Internett-utskrift er satt til Forbudt, er bare webbasert utskriftsstyring aktivert. Brukere kan ikke koble seg til delte skrivere gjennom Internett-utskrift.
  - Installasjoner av IIS på Windows Server 2008 er som standard ikke berørt av dette sikkerhetsproblemet. En datamaskin som kjører Windows Server 2008, kan fungere som en klientdatamaskin som bruker Internett-utskrift. Brukere av datamaskinen som foretar utskriftsforespørsler, må bli godkjent av utskriftsserveren før de kan bruke skriverne som er koblet til den serveren. For å gi en datamaskin som kjører Windows Server 2008, tillatelse til å fungere som server med støtte for Internett-utskrift, må du installere tjenester for webserver-rollen (IIS), utskriftstjenesterollen og Internett-utskriftsrollen i utskriftstjenesterollen.
  - For å kunne utnytte dette sikkerhetsproblemet må IIS-serveren ha utgående kommunikasjon med en skadelig datamaskin i port 445. Administratorer som blokkerer utgående SMB-tilkoblinger fra IIS-serveren til Internett, kan forhindre at dette sikkerhetsproblemet blir utnyttet på upålitelige systemer.

#### Midlertidige løsninger for sikkerhetsproblemet med overflyt av heltall i IPP-tjenesten – CVE-2008-1446

Midlertidige løsninger viser til en innstilling eller konfigurasjonsendring som ikke løser det underliggende sikkerhetsproblemet, men hjelper til med å blokkere kjente angrepsvektorer før du installerer oppdateringen. Microsoft har testet følgende midlertidige løsninger og informerer i diskusjonen om en midlertidig løsning reduserer funksjonaliteten:

  - **Deaktivere IPP-tjenesten**
    
    Slik deaktiverer du IPP-tjenesten i IIS 5.0 og IIS 5.1
    
    1.  Åpne **Internett-tjenestebehandling**.
    2.  Høyreklikk det **standard webområdet**, og velg **Egenskaper**.
    3.  Gå til kategorien **Startmappe**, og velg **Konfigurasjon**.
    4.  Rull ned for å finne filtypen **.printer** og den tilhørende banen til **msw3prt.dll**.
    5.  Klikk **Fjern**.
    6.  Klikk **OK**.
    7.  Start IIS-tjenesten på nytt.
    
    **Obs\!** Internett-utskrift kan konfigureres via gruppepolicy og via Internett-tjenestebehandling. Hvis det er en konflikt mellom gruppepolicyinnstillingene og innstillingene i Internett-tjenestebehandling, er det gruppepolicyinnstillingene som har forrang. Hvis du fjerner Internett-utskrift via Internett-tjenestebehandling, må du kontrollere at det ikke reaktiveres av enten lokale gruppepolicyer eller domenegruppepolicyer. (Standard gruppepolicy verken aktiverer eller deaktiverer Internett-utskrift.) I snapin-modulen for MMC-gruppepolicyen merker du av for Datamaskinkonfigurasjon | Administrative maler | Utskrift | Webbasert utskrift.
    
    Slik deaktiverer du IPP-tjenesten i IIS 6.0
    
    1.  Start **IIS-behandling** eller **IIS-snapin-modulen**.
    2.  Utvid **\* server\_navn**, der server\_navn er navnet på serveren.
    3.  Klikk **Webtjenesteutvidelser**.
    4.  Klikk **Internett-utskrift** i høyre rute, og klikk deretter **Forbudt**.
    5.  Avslutt IIS-behandling.
    
    Slik deaktiverer du IPP-tjenesten i IIS 7.0 i Windows Server 2008
    
    1.  Hvis **Serverbehandling** ikke kjører, klikker du **Start**, klikker **Administrative verktøy** og klikker deretter **Serverbehandling**. (Hvis dialogboksen **Brukerkontokontroll** vises, bekrefter du at handlingen den viser, er det du ønsker. Klikk deretter **Fortsett**.)
    2.  Utvid noden Roller, og høyreklikk deretter **Utskriftstjenester** og velg å **fjerne rolletjenester**.
    3.  Veiviseren for å fjerne rolletjenester vises. Hvis det er merket av for alternativet **Internett-utskrift**, fjerner du avmerkingen, klikker **Neste** og deretter **Fjern** for å fullføre veiviseren.
    
    **Hva løsningen kan føre til:** Brukere vil ikke kunne koble seg til delte skrivere gjennom Internett-utskrift.
    
    **Slik angrer du den midlertidige løsningen:**
    
    Slik aktiverer du IPP-tjenesten i IIS 5.0 og IIS 5.1
    
    1.  Åpne **Internett-tjenestebehandling**.
    2.  Høyreklikk det **standard webområdet**, og velg **Egenskaper**.
    3.  Gå til kategorien **Startmappe**, og velg **Konfigurasjon**.
    4.  Rull ned for å finne filtypen **.printer** og den tilhørende banen til **msw3prt.dll**.
    5.  Klikk **Legg til**.
    6.  Skriv inn følgende i feltet **Kjørbar**:  
        %systemroot%\\system32\\msw3prt.dll
    7.  Skriv inn følgende i feltet **Filtype**:  
        .printer
    8.  Skriv inn følgende i feltet **Begrens til**:  
        GET,POST
    9.  Etter at du har gjort dette, starter du IIS-tjenesten på nytt.
    
    Slik aktiverer IPP-tjenesten i IIS 6.0

#### Vanlige spørsmål om sikkerhetsproblemet med overflyt av heltall i IPP-tjenesten – CVE-2008-1446

**Hva er omfanget av sikkerhetsproblemet?**    
Dette er et sikkerhetsproblem som kan forårsake ekstern kjøring av kode. En angriper som klarer å utnytte dette sikkerhetsproblemet, kan få full kontroll over et berørt system. En angriper kan deretter installere programmer, vise, endre eller slette data eller opprette nye kontoer.

**Hva forårsaker sikkerhetsproblemet?**    
IPP-funksjonen (Internet Printing Protocol) blir implementert som en ISAPI-utvidelse for IIS. Det finnes et eksternt, erkjent sikkerhetsproblem i Microsoft-implementeringen av IPP-funksjonen på Windows-servere som kjører IIS. Det finnes en overflyt av heltall i implementeringen av IPP som kan tillate at en godkjent angriper kjører ekstern kode på berørte IIS-servere.

**Hva er ISAPI?**  
ISAPI (Internet Services Application Programming Interface) er en type teknologi som lar webutviklere utvide funksjonaliteten til webserverne sine ved å skrive spesiallaget kode, som tilbyr nye tjenester for en webserver. Den spesiallagde koden kan enten implementeres i et ISAPI-filter, hvis den nye funksjonaliteten tilbyr tjenester på lavt nivå, eller en ISAPI-utvidelse, dersom den nye funksjonaliteten tilbyr en tjeneste på høyt nivå. I dette tilfellet er den berørte koden en ISAPI-utvidelse.

**Hva slags ISAPI-utvidelse er det snakk om?**  
Den berørte ISAPI-utvidelsen er en som implementerer Internett-utskriftsprotokollen (IPP), en industristandard som er definert i RFCs 2910 og 2911. Med IPP kan du gi forespørsler om utskriftstjenester og vise statusen på utskriftsjobber over Internett via HTTP. For eksempel kan en reisende bruker sende en utskriftsjobb over Internett via IPP til nettverksskriveren på kontoret sitt. Brukeren kan også finne ut om utskriften ble gjennomført uten problemer.

I Microsoft Windows 2000 ble støtte for Internett-utskrift i operativsystemet introdusert. Med implementeringen i Microsoft Windows 2000 kan brukere skrive ut direkte til en URL-adresse og vise informasjon om utskriftsjobber via weblesere. Støtte for Internett-utskrift er aktivert som standard i serverversjonene av Microsoft Windows 2000. Det kan også installeres som en valgfri komponent i Microsoft Windows 2000 Professional, Windows XP, Windows Server 2003 og Windows Server 2008.

**Hvordan kan en angriper utnytte dette sikkerhetsproblemet?**    
En angriper som klarer å utnytte dette sikkerhetsproblemet, kan kjøre vilkårlig kode på systemet til en bruker med systemrettigheter. En angriper som klarer å utnytte dette sikkerhetsproblemet, kan få full kontroll over et berørt system. En angriper kan deretter installere programmer, vise, endre eller slette data eller opprette nye kontoer.

**Hvordan kan en angriper utnytte sikkerhetsproblemet?**    
En angriper kan sende en spesiallaget HTTP POST-forespørsel til en webserver med et sikkerhetsproblem. Dette kan føre til at serveren via IPP kobler seg til en maskin som er kontrollert av angriperen, og som fungerer som en skriver. Angriperen kan deretter bruke spesiallagde svar for å kjøre ekstern kode på den utvalgte serveren.

**Hvilke systemer er hovedsakelig i faresonen på grunn av sikkerhetsproblemet?**    
Det er hovedsakelig arbeidsstasjoner og servere som kjører IIS med IPP-tjenesten (Internett-utskriftsprotokoll) aktivert som er i faresonen på grunn av dette sikkerhetsproblemet.

**Hva gjør oppdateringen?**    
Denne sikkerhetsoppdateringen løser sikkerhetsproblemet ved å validere dataene som blir mottatt fra eksterne maskiner innenfor IPP-tjenesten.

**Var dette sikkerhetsproblemet offentliggjort da denne sikkerhetsbulletinen ble publisert?**    
Nei. Microsoft mottok informasjon om dette sikkerhetsproblemet gjennom begrenset offentliggjøring. Microsoft hadde ikke mottatt informasjon som tyder på at dette sikkerhetsproblemet var offentliggjort da denne sikkerhetsbulletinen opprinnelig ble publisert.

**Hadde Microsoft mottatt rapporter om at dette sikkerhetsproblemet ble utnyttet, da denne sikkerhetsbulletinen ble publisert?**    
Ja, Microsoft kjenner til målrettede angrep som forsøker å utnytte dette sikkerhetsproblemet. Eksempler på konseptbeviskode var imidlertid ikke publisert da denne sikkerhetsbulletinen først ble utgitt.

**Kontrollere at oppdateringen er installert**

  - **Microsoft Baseline Security Analyzer**
    
    Det kan være at du kan bruke MBSA-verktøyet (Microsoft Baseline Security Analyzer) til å kontrollere at en sikkerhetsoppdatering er aktivert på et berørt system. Se delen **Veiledning og verktøy for søking og distribusjon** tidligere i denne bulletinen hvis du vil ha mer informasjon.

  - **Kontroll av filversjon**
    
    Ettersom det finnes flere versjoner av Microsoft Windows, kan fremgangsmåten være annerledes for datamaskinen du bruker. I så fall ser du produktdokumentasjonen for å utføre trinnene.
    
    1.  Klikk **Start**, og klikk deretter **Søk**.
    2.  I ruten **Søkeresultater** klikker du **Alle filer og mapper** under **Søkehjelperen**.
    3.  I boksen **Hele eller deler av filnavnet** skriver du inn et filnavn fra den riktige filinformasjonstabellen og klikker deretter **Søk**.
    4.  I fillisten høyreklikker du et filnavn fra den riktige filinformasjonstabellen. Deretter klikker du **Egenskaper**.  
          
        **Obs\!** Avhengig av operativsystemversjonen eller programmene som er installert på systemet, kan det hende at noen av filene som er oppført i filinformasjonstabellen, ikke er installert.
    5.  I kategorien **Versjon** finner du versjonen av filen som er installert på systemet, ved å sammenligne den med versjonen i den respektive filinformasjonstabellen.  
          
        **Obs\!** Andre attributter enn filversjonen kan bli endret under installasjonen. Sammenligning av andre filattributter med informasjonen i filinformasjonstabellen er ikke en metode som støttes for å kontrollere at oppdateringen er aktivert. I noen tilfeller kan det også hende filer får nytt navn under installasjonen. Hvis filen eller versjonsinformasjonen mangler, bruker du en av de andre tilgjengelige metodene for å kontrollere oppdateringsinstallasjonen.

  - **Kontroll av registernøkkel**
    
    Det kan hende at du også kan kontrollere filene som installeres med denne sikkerhetsoppdateringen, ved å se på registernøklene i **Referansetabellen** i denne delen.
    
    Det er ikke sikkert at disse registernøklene inneholder en fullstendig liste over alle installerte filer. Det er heller ikke sikkert at disse registernøklene opprettes på riktig måte når en administrator eller OEM integrerer sikkerhetsoppdateringen i kildefilene for Windows-installasjonen.
