# Microsoft Security Bulletin MS08-047 - Important

## Sikkerhetsproblem med behandling av IPsec-policy kan føre til tilgjengeliggjøring av informasjon (953733)

Publisert: 12. august 2008 | Oppdatert: 13. august 2008

**Versjon:** 1.1

### Generell informasjon

#### Kortfattet sammendrag

Denne oppdateringen løser et personlig rapportert sikkerhetsproblem i måten som enkelte regler for Windows Internet Protocol-sikkerhet (IPsec) brukes. Dette sikkerhetsproblemet kan føre til at systemer ignorerer IPsec-policyer og overfører nettverkstrafikk i klartekst. Det kan igjen føre til at informasjon som skulle vært kryptert i et nettverk, tilgjengeliggjøres. En angriper som ser på trafikken i nettverket, kan vise og muligens endre innholdet i trafikken. Legg merke til at dette sikkerhetsproblemet ikke gjør det mulig for en angriper å kjøre kode eller utvide egne brukerrettigheter direkte. Det kan brukes til å samle inn nyttig informasjon som senere kan brukes til å skade det berørte systemet eller nettverket ytterligere.

Denne oppdateringen har alvorlighetsgraden Viktig for alle støttede versjoner av Windows Vista and Windows Server 2008. Hvis du vil ha mer informasjon, kan du se avsnittet **Berørt og uberørt programvare** i denne delen**.**

Sikkerhetsoppdateringen løser sikkerhetsproblemet ved å sikre at IPsec-regler behandles på riktig måte. Hvis du vil ha mer informasjon om sikkerhetsproblemet, kan du se avsnittet Vanlige spørsmål for den bestemte sikkerhetsproblemoppføringen under neste del, **Informasjon om sikkerhetsproblemer**.

**Anbefaling.** Microsoft anbefaler at kundene installerer oppdateringen så snart det er mulig.

**Kjente problemer.** Ingen

## Sikkerhetsproblem med IPsec-policy som kan føre til tilgjengeliggjøring av informasjon – CVE-2008-2246

Det finnes et viktig sikkerhetsproblem som kan føre til tilgjengeliggjøring av informasjon, i måten som IPsec-policyer importeres til Windows Server 2008-domener fra Windows Server 2003-domener på. Dette sikkerhetsproblemet kan føre til at systemer ignorerer IPsec-policyer og overfører nettverkstrafikk i klartekst. Det kan igjen potensielt føre til at informasjon som skulle vært kryptert i et nettverk, tilgjengeliggjøres. En angriper som fanger opp trafikken i nettverket, kan vise og muligens endre innholdet i trafikken. Legg merke til at dette sikkerhetsproblemet ikke gjør det mulig for en angriper å kjøre kode eller utvide egne brukerrettigheter direkte. Det kan brukes til å samle inn nyttig informasjon som senere kan brukes til å skade det berørte systemet ytterligere.

Hvis du vil se dette sikkerhetsproblemet som en standardoppføring i listen Common Vulnerabilities and Exposures, kan du se [CVE-2008-2246](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2246).

#### Begrensende faktorer for sikkerhetsproblem med IPsec-policy som kan føre til tilgjengeliggjøring av informasjon – CVE-2008-2246

Begrensning viser til en innstilling, vanlig konfigurering eller generell god fremgangsmåte som finnes i standardtilstanden og kan redusere alvorlighetsgraden til utnyttelsen av et sikkerhetsproblem. Følgende begrensende faktorer kan være nyttige i din situasjon:

  - I Windows Vista og Windows Server 2008 aktiveres ikke standardsvarregelen automatisk. Standardsvarregelen aktiveres bare automatisk når du oppretter en IPsec-policy via tidligere versjoner av Windows enn Windows Vista og Windows Server 2008.

#### Midlertidige løsninger for sikkerhetsproblem med IPsec-policy som kan føre til tilgjengeliggjøring av informasjon – CVE-2008-2246

Midlertidige løsninger viser til en innstilling eller konfigurasjonsendring som ikke løser det underliggende sikkerhetsproblemet, men hjelper til med å blokkere kjente angrepsvektorer før du installerer oppdateringen. Microsoft har testet følgende midlertidige løsninger og informerer i diskusjonen om en midlertidig løsning reduserer funksjonaliteten:

  - Ikke merk av for Standardsvarregel under opprettelse av en IPsec-policy eller fjern denne regelen fra eksisterende policyer siden denne regelen ikke gyldig i Windows Vista og Windows Server 2008 mer, og bare er aktuell på tidligere versjoner av Windows. Se [Microsoft Knowledge Base-artikkel 942964](http://support.microsoft.com/kb/942964/) for å få informasjon om hvordan du etterligner denne regelen i Windows Vista og Windows Server 2008

#### Vanlige spørsmål om sikkerhetsproblem med IPsec-policy som kan føre til tilgjengeliggjøring av informasjon – CVE-2008-2246

**Hva er omfanget av sikkerhetsproblemet?**    
Dette er et sikkerhetsproblem som kan føre til tilgjengeliggjøring av informasjon. En angriper som utnytter dette sikkerhetsproblemet, kan lese innholdet i nettverkstrafikk som ellers ville vært kryptert. Legg merke til at dette sikkerhetsproblemet ikke gjør det mulig for en angriper å kjøre kode eller utvide egne brukerrettigheter direkte. Det kan brukes til å samle inn nyttig informasjon som senere kan brukes til å skade det berørte systemet ytterligere.

**Hva forårsaker sikkerhetsproblemet?**    
Dette sikkerhetsproblemet har oppstått fra en feil når IPsec-standardpolicyen importeres fra et Windows Server 2003-domene til et Windows Server 2008-domene. I visse situasjoner kan denne feilen føre til at alle IPsec-regler ignoreres.

**Hva er IPsec?**    
Internet Protocol-sikkerhet (IPsec) er et rammeverk med åpne standarder for sikring av privat, sikker kommunikasjon over IP-nettverk (Internet Protocol) gjennom bruk av kryptografiske sikkerhetstjenester. IPsec-arbeidsgruppen Internet Engineering Task Force (IETF) definerer IPsec-standardene.

IPsec er den langsiktige veien å gå for sikker nettverkstrafikk. Det gir omfattende beskyttelse mot private nettverk og Internett-angrep via ende-til-ende-sikkerhet. De eneste datamaskinene som må kjenne til IPsec-beskyttelsen, er senderen og mottakeren i kommunikasjonen. IPsec gir mulighet til å beskytte kommunikasjon mellom arbeidsgrupper, datamaskiner i lokalnett, domeneklienter og -servere, avdelingskontorer (som kan være på et annet fysisk sted), ekstranett og mobile klienter. Hvis du vil ha mer informasjon om IPsec, kan du se [Microsofts webområde](http://www.microsoft.com/technet/network/ipsec/ipsecfaq.mspx).

**Hvordan kan en angriper utnytte dette sikkerhetsproblemet?**    
En angriper som utnytter dette sikkerhetsproblemet, kan se, og i noen tilfeller endre, nettverkstrafikken som skulle vært kryptert, i klartekst.

**Hvordan kan en angriper utnytte sikkerhetsproblemet?**    
Dette sikkerhetsproblemet kan føre til at en sikkerhetsfunksjon ikke fungerer som den skal. For å kunne utnytte dette sikkerhetsproblemet må en angriper overvåke nettverkstrafikken. Angriperen kan lese og muligens endre innholdet i nettverkstrafikken som han/hum ser fra sitt "utsiktspunkt". Under normale forhold beskytter IPsec-policyer mot at denne trafikken skal fanges opp, ved å kryptere innholdet. En angriper kan ikke få en slik situasjon til å oppstå med mindre han/hun tidligere anskaffet seg administrative tillatelser til domenekontrolleren. I stedet avhenger de av at en administrator uten å vite det feilkonfigurerer IPsec-regelsettet og fører til at informasjon overføres i klartekst.

**Hvilke systemer er hovedsakelig i faresonen på grunn av sikkerhetsproblemet?**    
Det er først og fremst Windows Vista-arbeidsstasjoner og Windows Server 2008-servere som er underlagt IPsec-policyer i et Windows Server 2008-domene som er oppgradert fra et Windows Server 2003-domene, som er i faresonen.

**Hva gjør oppdateringen?**    
Oppdateringen fjerner sikkerhetsproblemet ved å sikre at IPsec-regler behandles på riktig måte.

**Var dette sikkerhetsproblemet offentliggjort da denne sikkerhetsbulletinen ble publisert?**    
Nei. Microsoft mottok informasjon om dette sikkerhetsproblemet gjennom begrenset offentliggjøring. Microsoft hadde ikke mottatt informasjon som tyder på at dette sikkerhetsproblemet var offentliggjort da denne sikkerhetsbulletinen opprinnelig ble publisert.

**Hadde Microsoft mottatt rapporter om at dette sikkerhetsproblemet ble utnyttet, da denne sikkerhetsbulletinen ble publisert?**    
Nei. Microsoft hadde ikke mottatt informasjon som tydet på at dette sikkerhetsproblemet offentlig hadde blitt brukt til å angripe kunder, og hadde ikke sett noen eksempler på at konseptbeviskode var publisert da denne sikkerhetsbulletinen først ble publisert.
